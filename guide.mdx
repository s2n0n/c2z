## How to Run on Kubernetes

### 1ï¸âƒ£ Automated Installation (Recommended)

The simplest method is to use the provided `install.sh` script.

```bash
# Run from project root
chmod +x install.sh
./install.sh
```

**What the installation script automatically performs:**

1. **Python Virtual Environment Setup** (`.venv`)
   - Create Python 3 virtual environment
   - Install dependencies from `requirements.txt` (click, tabulate, pyyaml)

2. **System Requirements Verification**
   - Verify RAM 16GB or more
   - Verify Docker installation (required for macOS)

3. **Kubernetes Cluster Installation**
   - **Linux**: K3s (native installation)
   - **macOS**: k3d (Docker Desktop based)
   
4. **Helm Installation**
   - Automatic Helm 3 installation

5. **c2z Stack Deployment**
   - Deploy `charts/c2z` Helm chart to `c2z-system` namespace
   - Install all enabled scenarios and monitoring stack

6. **CLI Wrapper Script Creation**
   - Create `./c2z` executable (automatically activates Python virtual environment)

### 2ï¸âƒ£ Manual Installation

If you want to execute each step individually:

#### Step 1: Kubernetes Cluster Setup

**Linux (K3s):**
```bash
curl -sfL https://get.k3s.io | sh -s - \
  --write-kubeconfig-mode 644 \
  --disable traefik \
  --disable servicelb

# Configure kubeconfig
mkdir -p ~/.kube
sudo cp /etc/rancher/k3s/k3s.yaml ~/.kube/config
sudo chown $USER:$USER ~/.kube/config
```

**macOS (k3d):**
```bash
# Install k3d
curl -s https://raw.githubusercontent.com/k3d-io/k3d/main/install.sh | bash

# Create cluster
k3d cluster create c2z \
  --api-port 6443 \
  --port "80:80@server:0" \
  --port "443:443@server:0" \
  --port "3000-3005:3000-3005@server:0" \
  --agents 1 \
  --no-lb \
  --k3s-arg "--disable=traefik@server:0" \
  --k3s-arg "--disable=servicelb@server:0" \
  --wait

# Configure kubeconfig
k3d kubeconfig merge c2z --kubeconfig-switch-context
```

#### Step 2: Helm Installation

```bash
curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash
```

#### Step 3: c2z Deployment

```bash
# Deploy with local chart
helm upgrade --install c2z ./charts/c2z \
  --namespace c2z-system \
  --create-namespace \
  --wait \
  --timeout 10m
```

#### Step 4: CLI Setup

```bash
# Create and activate Python virtual environment
python3 -m venv .venv
source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Use CLI
./c2z list
```

### 3ï¸âƒ£ CLI Usage

Once installation is complete, you can manage scenarios using the `c2z` CLI.

```bash
# List available scenarios
./c2z list

# Deploy scenario (activate)
./c2z deploy web-vuln

# Delete scenario (deactivate)
./c2z delete web-vuln

# Check overall system status
./c2z status

# View scenario logs
./c2z logs web-vuln
```

### 4ï¸âƒ£ Deployment Verification

```bash
# Check Pods in all namespaces
kubectl get pods -A

# Check c2z-system namespace
kubectl get all -n c2z-system

# Check scenario namespace
kubectl get all -n scenario-web-vuln
```

### 5ï¸âƒ£ Service Access

Once scenarios are deployed, you can access them as follows:

```bash
# Access via port-forward
kubectl port-forward -n scenario-web-vuln svc/dvwa 8080:80
# Access http://localhost:8080 in browser

kubectl port-forward -n scenario-web-vuln svc/juice-shop 3000:3000
# Access http://localhost:3000 in browser

# Access Kali Linux attacker environment
kubectl exec -it -n c2z-system deployment/kali -- /bin/bash
```

## Key Component Roles

### ğŸ”§ Installation and Configuration Files

#### `install.sh`
- **Role**: Main script that automatically installs the entire c2z environment
- **Key Features**:
  - Python virtual environment setup
  - System requirements verification (RAM, Docker)
  - Kubernetes cluster installation (OS-specific: K3s/k3d)
  - Helm installation
  - c2z Helm chart deployment
  - CLI wrapper script creation
- **Execution Flow**:
  1. `setup_venv()` - Prepare Python environment
  2. `check_requirements()` - System verification
  3. `install_k8s()` - Kubernetes installation
  4. `install_helm()` - Helm installation
  5. `deploy_c2z()` - Chart deployment
  6. `create_wrapper()` - CLI wrapper creation

#### `requirements.txt`
- **Role**: Define dependencies for Python CLI tool
- **Packages**:
  - `click`: CLI framework
  - `tabulate`: Table format output
  - `pyyaml`: YAML parsing (Kubernetes resource handling)

#### `c2z-cli.py`
- **Role**: Python CLI tool for scenario management
- **Main Commands**:
  - `list`: Display available scenario list
  - `deploy <scenario-id>`: Activate scenario (update Helm values)
  - `delete <scenario-id>`: Deactivate scenario
  - `status`: Check overall system status
  - `logs <scenario-id>`: View scenario logs
- **Operation Method**:
  - Uses monolithic Helm chart structure
  - Toggle scenarios via `helm upgrade --reuse-values --set scenarios.<name>.enabled=true/false`
  - Status check and log viewing through `kubectl` commands

#### `c2z` (Wrapper Script)
- **Role**: Automatically activate Python virtual environment and execute `c2z-cli.py`
- **Creation**: Auto-generated by `install.sh`
- **Content**:
  ```bash
  #!/bin/bash
  source $(dirname "$0")/.venv/bin/activate
  python3 $(dirname "$0")/c2z-cli.py "$@"
  ```

### Helm Chart Structure

#### `charts/c2z/Chart.yaml`
- **Role**: Define Helm chart metadata
- **Content**:
  - Chart name: `c2z`
  - Version: `0.1.0`
  - App version: `1.0.0`
  - Type: `application`

#### `charts/c2z/values.yaml`
- **Role**: Define default configuration values for the chart
- **Main Sections**:
  - `global`: Common settings (image registry, Pull Policy)
  - `attacker`: Kali Linux attacker environment configuration
    - Image, resource limits, persistent storage settings
  - `scenarios`: Scenario-specific activation flags and settings
    - `webVuln.enabled`: Web vulnerability scenario activation
    - `containerEscape.enabled`: Container Escape scenario
    - `networkAttack.enabled`: Network Attack scenario
  - `monitoring`: Prometheus, Grafana, Loki configuration
  - `networkPolicies`: Network isolation policy activation

#### `charts/c2z/templates/namespaces.yaml`
- **Role**: Create Kubernetes namespaces
- **Created Namespaces**:
  - `c2z-system`: Attacker environment and monitoring stack
  - `scenario-web-vuln`: Web vulnerability scenario
  - `scenario-container-escape`: Container Escape scenario
  - `scenario-network-attack`: Network Attack scenario

#### `charts/c2z/templates/network-policies.yaml`
- **Role**: Define network isolation policies
- **Security Features**:
  - Network isolation between scenarios
  - Default Deny policy (only explicitly allowed communication)
  - Allow access from attacker environment to scenarios

#### `charts/c2z/templates/resource-quotas.yaml`
- **Role**: Resource limits per namespace
- **Limit Items**:
  - CPU, memory requests/limits
  - Pod count limits
  - Prevent resource abuse

### Attacker Environment (Attacker Zone)

#### `attacker-zone/kali-deployment.yaml`
- **Role**: Deploy Kali Linux attacker Pod
- **Features**:
  - Image: `kalilinux/kali-rolling:latest`
  - Resources: 2Gi RAM, 1 vCPU (request) / 4Gi RAM, 2 vCPU (limit)
  - Persistent storage mount (`/root`)
  - Pre-installed network tools and penetration testing tools

#### `attacker-zone/kali-pvc.yaml`
- **Role**: Persistent storage volume claim for Kali environment
- **Capacity**: 20Gi
- **Purpose**: Store attack scripts and results

#### `attacker-zone/kali-service.yaml`
- **Role**: Expose service for Kali Pod
- **Type**: ClusterIP (internal access)

## Troubleshooting

### Issue: Pod in Pending State
```bash
# Check cause
kubectl describe pod <pod-name> -n <namespace>

# Adjust resource limits in values.yaml if resources are insufficient
helm upgrade c2z ./charts/c2z --set attacker.kali.resources.requests.memory=1Gi
```

### Issue: Service Access Failure
```bash
# Check service status
kubectl get svc -n scenario-web-vuln

# Set up port-forward
kubectl port-forward -n scenario-web-vuln svc/dvwa 8080:80
```

### Issue: Helm Deployment Failure
```bash
# Check Helm release status
helm list -n c2z-system

# Rollback
helm rollback c2z -n c2z-system

# Complete reinstallation
helm uninstall c2z -n c2z-system
./install.sh
```
